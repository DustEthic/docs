# Politique de sécurité DustEthic

Date : 2026-06-13

## Statut

DustEthic est en Phase 0. Aucun contrat, script ou flux n'est recommandé en production.

## Ne pas utiliser en production

Tout code, schéma ou exemple publié avant audit doit être considéré comme expérimental.

## Signaler un risque

Pour signaler un problème de sécurité ou un risque sérieux :

- contacte le projet via le formulaire du site : <https://dustethic.org/contact.html> ;
- évite de publier publiquement des détails exploitables avant échange ;
- indique le périmètre, l'impact, les étapes de reproduction et les sources.

## Périmètre de vigilance

- permissions wallet trop larges ;
- signatures incompréhensibles ;
- autorisations non révocables ;
- contrats avec rôle d'administration opaque ;
- frais masqués ;
- actifs piégés ou non standards ;
- preuve de lot falsifiable ou incomplète ;
- confusion entre intention signée et don exécuté ;
- promesse juridique ou fiscale non validée.

## Principe

Une UX qui pousse l'utilisateur à signer sans comprendre est un problème de sécurité, même si le code fonctionne techniquement.
