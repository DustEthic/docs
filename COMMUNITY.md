# Communauté DustEthic

Date : 2026-06-13

## Objectif

Construire une communauté utile avant de construire une technologie risquée.

DustEthic a besoin de critiques techniques, juridiques, UX, ONG et sécurité. Le Discord doit servir à transformer les idées vagues en propositions testables, puis en issues GitHub.

## Salons Discord recommandés

- `📜-règles` : règles, statut réel, interdictions.
- `🤝-présentations` : profils et compétences.
- `📣-annonces` : annonces courtes.
- `🌱-le-concept` : questions générales.
- `❓-faq` : questions récurrentes.
- `💡-idées` : idées au format problème / solution / risques / preuve.
- `🧩-standard` : format d'intention, preuves, frais, versionnement.
- `🛠️-technique` : wallets, paymasters, contrats, testnets, specs.
- `🧾-preuves-transparence` : preuve de lot lisible et vérifiable.
- `⚖️-juridique-fiscalité` : points prudentiels.
- `🧡-ong` : contraintes ONG.
- `👛-wallets` : intégration wallet.
- `🔁-relayeurs` : opérateurs, frais, logs, non-custodial.
- `💬-discussions-général` : discussion ouverte.
- `🎮-off-topic` : détente.

## Règles de base

1. Pas de promotion de token, NFT, airdrop ou investissement.
2. Pas de promesse de rendement ou d'impact non prouvé.
3. Pas de collecte active par DustEthic.
4. Toute affirmation technique importante doit être sourcée.
5. Les critiques argumentées sont bienvenues.
6. Ne jamais publier de clé privée, seed phrase, signature sensible ou donnée personnelle.
7. Une idée mûre doit devenir une issue GitHub.

## Format d'idée utile

```text
Problème :
Solution proposée :
Hypothèses :
Risques :
Preuve ou test nécessaire :
Sources :
```

## Ton public

- Factuel.
- Prudent.
- Vérifiable.
- Accessible aux non-techniciens.
- Transparent sur les limites.

## À éviter

- “C'est déjà prêt.”
- “C'est gratuit.”
- “C'est sans risque.”
- “On va révolutionner les dons.”
- “Il suffit d'utiliser tel protocole.”

Le bon message : DustEthic cherche à construire un standard ouvert, critiquable, vérifiable et légalement prudent.
